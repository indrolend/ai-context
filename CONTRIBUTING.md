# Contributing to indrolend-website

Welcome! To keep this project healthy and collaborative, please follow these guidelines:

## Workflow
- Read and follow AI_WORKFLOW_BEST_PRACTICES.md for philosophy and process.
- Use feature branches for new work. Submit pull requests for review.
- Reference relevant issues in your PRs.

## Code Standards
- Follow the repo's STYLE_GUIDE.md and CODE_COMMENTS.md.
- Write clear, maintainable code. Add comments for non-obvious logic.
- Use consistent naming and folder structure.

## Testing
- All new features and bugfixes must include automated tests.
- Place tests in the `tests/` folder or alongside the code (if appropriate).

## Linting & Formatting
- Run `npm run lint` and `npm run format` before pushing.
- Fix all linter and formatter errors.

## Documentation
- Update or add README.md/index.md in any folder you touch.
- Document new features, APIs, or scripts.

## Legacy & Dead Code
- Mark legacy code clearly. Archive or delete unused scripts/features.

## Ownership
- If you own a feature, document it in the relevant README or a CODEOWNERS file.

## Automation
- All checks must pass in CI before merging.

For more details, see:

Thank you for contributing!
