# Ownership and Feature Boundaries

To clarify code ownership and responsibility:

- Each major feature or module should have an owner (person or team).
- Document ownership in the relevant README.md or in a CODEOWNERS file.
- For shared code, list maintainers or reviewers.

---

Example CODEOWNERS file:

```
# Set up GitHub code owners for PR review
/js/spa/ @your-github-username
/backend/ @backend-team
```

---

Update this file as your team grows!
